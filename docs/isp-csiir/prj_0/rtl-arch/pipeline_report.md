## 2. 各阶段组合逻辑复杂度分析

### 2.1 Stage 1: 梯度计算与窗口大小确定

#### 2.1.1 运算分解

**Sobel 卷积运算:**
```
grad_h = row0_sum - row4_sum
其中:
  row0_sum = win[0][0] + win[0][1] + win[0][2] + win[0][3] + win[0][4]
  row4_sum = win[4][0] + win[4][1] + win[4][2] + win[4][3] + win[4][4]
```

**组合逻辑分析:**

| 运算 | 加法数量 | 加法器位宽 | 使用平衡树 | FO4 估算 |
|------|----------|-----------|-----------|----------|
| row_sum (5 个) | 5 × 4 = 20 | 10-bit → 13-bit | 3 级 | 18-24 |
| grad_h 差分 | 1 | 13-bit | 1 级 | 4-5 |
| 绝对值 | 1 (条件取反) | 14-bit | 1 级 | 3-4 |
| grad 求和 | 1 | 14-bit | 1 级 | 4-5 |
| grad_max (3 选 1) | 2 | 14-bit 比较 | 2 级 | 12-16 |
| win_size LUT | 4 | 14-bit 比较 | 4 级 | 16-24 |

**总 FO4 (不拆分): ~57-78 FO4** — 接近临界值

#### 2.1.2 流水线拆分方案

```
┌─────────────────────────────────────────────────────────────────────┐
│ Stage 1: 5 cycles                                                    │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────┤
│ Cycle 0     │ Cycle 1     │ Cycle 2     │ Cycle 3     │ Cycle 4     │
│ Sobel 卷积  │ 梯度差分    │ 绝对值求和  │ 梯度最大值  │ LUT 查表    │
│ ~20 FO4     │ ~8 FO4      │ ~12 FO4     │ ~15 FO4     │ ~15 FO4     │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┘
```

**拆分理由:**
1. Sobel 卷积 (20 FO4) 单独一级，保证时序裕量
2. 梯度最大值计算 (15 FO4) 单独一级，避免与 LUT 级联
3. 每级都有充足时序裕量 (>40 FO4)

### 2.2 Stage 2: 多尺度方向性平均

#### 2.2.1 运算分解

**核选择与加权求和:**
```
kernel_select = LUT(win_size_clip)
sum_c = Σ(window × weight_center)  // 最多 25 个乘累加
avg_c = sum_c / weight_c
```

**组合逻辑分析:**

| 运算 | 操作数量 | 位宽 | 平衡树深度 | FO4 估算 |
|------|----------|------|-----------|----------|
| 核选择 (比较链) | 4 | 6-bit | 4 级 | 8-12 |
| 25 个乘法 | 25 | 10-bit × 4-bit | - | 25-35 (并行) |
| 25 个加法 (求和) | 24 | 14-bit | 5 级 | 25-30 |
| 除法 (sum/weight) | 1 | 20-bit / 8-bit | - | 40-60 |

**除法器详细分析:**

| 除法器类型 | 周期数 | 单周期 FO4 | 适用性 |
|-----------|--------|-----------|--------|
| 组合逻辑除法 | 1 | 80-100 | 不适用 |
| 基数 2 迭代 | 20 | 8-10 | 太慢 |
| 基数 4 迭代 | 10 | 15-20 | 可接受 |
| 查表近似 | 1-2 | 20-30 | 推荐 |
| Newton-Raphson | 2-3 | 25-35 | 推荐 |

**总 FO4 (不拆分): ~98-137 FO4** — 严重超标

#### 2.2.2 流水线拆分方案

```
┌───────────────────────────────────────────────────────────────────────────────┐
│ Stage 2: 8 cycles                                                              │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────┬─────────┤
│ Cycle 0-3   │ Cycle 4     │ Cycle 5     │ Cycle 6     │ Cycle 7a    │ Cycle 7b│
│ 窗口延迟    │ 核选择      │ 加权求和 L1  │ 加权求和 L2  │ 除法 L1     │ 除法 L2 │
│ 对齐        │ ~10 FO4     │ ~35 FO4     │ ~20 FO4     │ ~25 FO4     │ ~20 FO4 │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┴─────────┘
```

**拆分理由:**
1. 窗口延迟 4 cycles: 与 Stage 1 的 win_size_clip 输出对齐
2. 加权求和拆分为 2 级: 乘法 + 部分加法 (Cycle 5)，剩余加法 (Cycle 6)
3. 除法拆分为 2 级: 使用 Newton-Raphson 迭代或查表近似

**除法实现方案:**
```verilog
// 使用倒数查表 + 乘法近似除法
// Cycle 7a: 查表得 1/weight (8-bit 地址，16-bit 结果)
// Cycle 7b: sum × (1/weight) → 右移归一化

reg [15:0] inv_weight_lut [0:255];  // 1/weight 查表
wire [15:0] inv_weight = inv_weight_lut[weight];
wire [35:0] product = sum × inv_weight;
wire [9:0]  avg = product[25:16];  // 取高 10 位
```

### 2.3 Stage 3: 梯度加权方向融合

#### 2.3.1 运算分解

**排序与加权融合:**
```
grad_sorted = sort_descending(grad_c, grad_u, grad_d, grad_l, grad_r)  // 5 数排序
blend_sum = Σ(avg × grad_sorted)  // 5 个乘累加
blend_avg = blend_sum / grad_sum
```

**排序网络分析:**

5 数排序的最优比较网络:
- 比较器数量: 7-9 个
- 比较深度: 5-6 级
- 每个 14-bit 比较器: ~6 FO4

**排序网络 FO4 估算:**
- 级联比较: 6 级 × 6 FO4 = 36 FO4

**加权乘累加分析:**

| 运算 | 操作数量 | 位宽 | 平衡树深度 | FO4 估算 |
|------|----------|------|-----------|----------|
| 5 个乘法 (avg × grad) | 5 | 10-bit × 14-bit | - | 30-35 (并行) |
| 5 个加法 | 4 | 24-bit | 3 级 | 12-15 |
| 梯度和 | 4 | 14-bit | 3 级 | 10-12 |
| 除法 | 1 | 26-bit / 17-bit | - | 50-70 |

**总 FO4 (不拆分): ~118-152 FO4** — 严重超标

#### 2.3.2 流水线拆分方案

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ Stage 3: 6 cycles                                                            │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────┬───────┤
│ Cycle 0     │ Cycle 1     │ Cycle 2     │ Cycle 3     │ Cycle 4     │ Cycle 5│
│ 输入缓冲    │ 排序 L1     │ 排序 L2     │ 加权乘积    │ 加权求和    │ 除法  │
│ ~10 FO4     │ ~20 FO4     │ ~15 FO4     │ ~35 FO4     │ ~20 FO4     │ ~30 FO4│
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┴───────┘
```

**拆分理由:**
1. 排序网络拆分为 2 级: 比较级 1-3 (Cycle 1)，比较级 4-6 (Cycle 2)
2. 加权乘积单独一级: 5 个并行乘法器
3. 加权求和单独一级: 平衡加法树
4. 除法使用查表近似: 类似 Stage 2

**排序网络流水化:**
```verilog
// Cycle 1: 比较 Pass 1-3
wire [13:0] s0_p1, s1_p1, s2_p1, s3_p1, s4_p1;  // Pass 1 结果
wire [13:0] s0_p3, s1_p3, s2_p3, s3_p3, s4_p3;  // Pass 3 结果

// Cycle 2: 比较 Pass 4-6 + 寄存
reg [13:0] grad_s0, grad_s1, grad_s2, grad_s3, grad_s4;
```

### 2.4 Stage 4: IIR 滤波与混合输出

#### 2.4.1 运算分解

**三级混合运算:**
```
// IIR 混合
blend_iir = (ratio × blend + (64-ratio) × avg_u) >> 6

// 窗混合
blend_out = (blend_iir × factor + center × (4-factor)) >> 2

// 最终混合
dout = (blend0_out × remainder + blend1_out × (8-remainder)) >> 3
```

**组合逻辑分析:**

| 运算 | 乘法数量 | 加法数量 | 位宽 | FO4 估算 |
|------|----------|----------|------|----------|
| ratio 选择 | 0 | 0 | 8-bit | 10-15 |
| IIR 混合 | 2 | 2 | 10-bit × 8-bit | 35-45 |
| 窗混合 | 2 | 2 | 10-bit × 2-bit | 25-30 |
| 最终混合 | 2 | 2 | 10-bit × 3-bit | 20-25 |

**总 FO4 (不拆分): ~90-115 FO4** — 超标

#### 2.4.2 流水线拆分方案

```
┌─────────────────────────────────────────────────────────────────────────┐
│ Stage 4: 5 cycles                                                        │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────────┤
│ Cycle 0     │ Cycle 1     │ Cycle 2     │ Cycle 3     │ Cycle 4         │
│ 输入缓冲    │ 比例选择    │ IIR 混合    │ 窗混合      │ 最终混合        │
│ ~5 FO4      │ ~10 FO4     │ ~35 FO4     │ ~25 FO4     │ ~20 FO4         │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────────┘
```

**拆分理由:**
1. IIR 混合单独一级: 包含 2 个乘法和加法
2. 窗混合单独一级: 包含 2 个乘法和加法
3. 最终混合单独一级: 简单的线性混合

## 4. 流水线边界选择依据

### 4.1 边界选择原则

1. **组合逻辑深度均衡**: 每级控制在 30-40 FO4 以内
2. **数据依赖最小化**: 减少跨级数据传递
3. **寄存器位宽优化**: 在位宽较窄处插入寄存器
4. **关键路径隔离**: 将复杂运算单独成级

### 4.2 各阶段边界选择详解

#### Stage 1 边界选择

```
边界 1 (Cycle 0 → 1): row_sum 计算后
  理由: 加法树输出位宽较宽 (13-bit)，适合插入寄存器

边界 2 (Cycle 1 → 2): grad_h/v 差分后
  理由: 绝对值操作需要条件逻辑，隔离差分结果

边界 3 (Cycle 2 → 3): grad 求和后
  理由: 梯度值已归一化，位宽可控

边界 4 (Cycle 3 → 4): grad_max 计算后
  理由: LUT 查表操作简单，可快速完成
```

#### Stage 2 边界选择

```
边界 1 (Cycle 0-3): 窗口延迟
  理由: 与 Stage 1 输出时序对齐

边界 2 (Cycle 4 → 5): 核选择后
  理由: 乘法输入稳定，减少毛刺

边界 3 (Cycle 5 → 6): 乘法完成后
  理由: 乘积位宽较宽 (14-bit)，适合流水

边界 4 (Cycle 6 → 7): 加法树完成后
  理由: 除法输入就绪
```

#### Stage 3 边界选择

```
边界 1 (Cycle 0 → 1): 梯度获取后
  理由: 行缓存读取需要 1 cycle

边界 2 (Cycle 1 → 2): 排序 Pass 1-3 后
  理由: 排序网络中间结果寄存

边界 3 (Cycle 2 → 3): 排序完成后
  理由: 乘法输入稳定

边界 4 (Cycle 3 → 4): 乘积完成后
  理由: 加法树输入就绪

边界 5 (Cycle 4 → 5): 加法完成后
  理由: 除法输入就绪
```

#### Stage 4 边界选择

```
边界 1 (Cycle 0 → 1): 输入寄存
  理由: IIR 行缓存读取对齐

边界 2 (Cycle 1 → 2): 比例选择后
  理由: 乘法输入稳定

边界 3 (Cycle 2 → 3): IIR 混合后
  理由: 输出稳定，可进行下一级混合

边界 4 (Cycle 3 → 4): 窗混合后
  理由: 最终混合输入就绪
```

## 5. 寄存器位宽优化策略

### 5.1 位宽优化原则

1. **在窄位宽处插入寄存器**: 减少寄存器数量
2. **避免过早扩展位宽**: 在必要时才扩展
3. **利用对称性**: 相同运算共享寄存器

### 5.2 各阶段位宽优化

#### Stage 1 位宽优化

| 流水级 | 寄存器位宽 | 总位宽 | 优化说明 |
|--------|-----------|--------|----------|
| Cycle 0 → 1 | row_sum[13], col_sum[13] × 10 | 143-bit | 加法树输出 |
| Cycle 1 → 2 | grad_raw[14] × 2 | 28-bit | 差分后 |
| Cycle 2 → 3 | grad_abs[14] × 2, grad[14] | 42-bit | 绝对值后 |
| Cycle 3 → 4 | grad_max[14] | 14-bit | 最大值后 |
| Cycle 4 输出 | grad[14], win_size[6] | 20-bit | 最终输出 |

**优化效果:**
- 原始方案: 每级存储完整窗口 (250-bit)
- 优化后: 仅存储中间结果 (平均 ~50-bit/级)

#### Stage 2 位宽优化

| 流水级 | 寄存器位宽 | 总位宽 | 优化说明 |
|--------|-----------|--------|----------|
| Cycle 0-3 | window[10] × 25 | 250-bit | 窗口延迟 (必需) |
| Cycle 4 → 5 | kernel_sel[3] | 3-bit | 核选择 |
| Cycle 5 → 6 | sum_partial[14] × 10 | 140-bit | 部分和 |
| Cycle 6 → 7 | sum[20] × 10, weight[8] × 10 | 280-bit | 完整和 |
| Cycle 7 输出 | avg[10] × 10 | 100-bit | 最终输出 |

**优化效果:**
- 使用部分和减少中间寄存器
- 除法后位宽压缩到 10-bit

#### Stage 3 位宽优化

| 流水级 | 寄存器位宽 | 总位宽 | 优化说明 |
|--------|-----------|--------|----------|
| Cycle 0 → 1 | avg[10] × 10, grad[14] × 5 | 170-bit | 输入 |
| Cycle 1 → 2 | grad_sorted[14] × 5 | 70-bit | 排序中间 |
| Cycle 2 → 3 | grad_sorted[14] × 5 | 70-bit | 排序完成 |
| Cycle 3 → 4 | blend_partial[24] × 10 | 240-bit | 乘积 |
| Cycle 4 → 5 | blend_sum[26] × 2, grad_sum[17] | 69-bit | 加法完成 |
| Cycle 5 输出 | blend_avg[10] × 2 | 20-bit | 最终输出 |

**优化效果:**
- 排序网络使用 14-bit 寄存器，避免扩展
- 加权乘积使用 24-bit，加权和 26-bit

#### Stage 4 位宽优化

| 流水级 | 寄存器位宽 | 总位宽 | 优化说明 |
|--------|-----------|--------|----------|
| Cycle 0 → 1 | blend_avg[10] × 2, avg_u[10] × 2 | 40-bit | 输入 |
| Cycle 1 → 2 | ratio[8], factor[3], remain[3] | 14-bit | 比例选择 |
| Cycle 2 → 3 | blend_iir[17] × 2 | 34-bit | IIR 混合 |
| Cycle 3 → 4 | blend_out[12] × 2 | 24-bit | 窗混合 |
| Cycle 4 输出 | dout[10] | 10-bit | 最终输出 |

**优化效果:**
- 三级混合逐级压缩位宽
- 最终输出仅 10-bit

### 5.3 总寄存器位宽对比

| 方案 | 总寄存器位宽 | 说明 |
|------|-------------|------|
| 未经优化 | ~15,000-bit | 每级存储完整数据 |
| 优化后 | ~8,000-bit | 仅存储必要中间结果 |
| 节省 | ~47% | 显著减少 |

## 6. IIR 反馈路径特殊处理

### 6.1 反馈路径时序分析

```
数据流时序:
┌─────────────────────────────────────────────────────────────────┐
│ Row N:                                                           │
│ din → ... → Stage2(avg_u_N) → 写IIR缓存 → Stage4(读avg_u_N-1)   │
│                           │                    │                 │
│                           └────────────────────┘                 │
│                              时序间隔: ~IMG_WIDTH cycles          │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 时序冲突分析

| 操作 | 时序位置 | 周期数 | 冲突风险 |
|------|----------|--------|----------|
| avg_u 写入 | Stage 2 Cycle 7 | N | 无 |
| avg_u 读取 | Stage 4 Cycle 0 | N + IMG_WIDTH | 无 |
| dout 写回 | Stage 4 Cycle 4 | N + IMG_WIDTH + 4 | 无 |

**结论:** 由于读写间隔大于 1 行时间 (~5472 cycles)，无时序冲突。

### 6.3 行缓存组织

```verilog
// IIR 行缓存双缓冲结构
reg [DATA_WIDTH-1:0] avg0_u_buf_0 [0:IMG_WIDTH-1];
reg [DATA_WIDTH-1:0] avg0_u_buf_1 [0:IMG_WIDTH-1];
reg [DATA_WIDTH-1:0] avg1_u_buf_0 [0:IMG_WIDTH-1];
reg [DATA_WIDTH-1:0] avg1_u_buf_1 [0:IMG_WIDTH-1];

wire buf_sel = row_cnt[0];  // 奇偶行切换

// 读: 上一行
wire [DATA_WIDTH-1:0] avg0_u_rd = buf_sel ? avg0_u_buf_0[pixel_x] : avg0_u_buf_1[pixel_x];

// 写: 当前行
always @(posedge clk) begin
    if (stage2_valid) begin
        if (buf_sel)
            avg0_u_buf_1[pixel_x] <= avg0_u;
        else
            avg0_u_buf_0[pixel_x] <= avg0_u;
    end
end
```

| 关键路径裕量 | 最小 35 FO4 |
| 寄存器位宽 | 已优化 |

### 8.2 后续优化建议

1. **使用 DSP 块**: Stage 2/3/4 的乘累加和除法可使用 DSP 块加速
2. **行缓存 SRAM 化**: 大容量行缓存使用 SRAM 减少面积
3. **时钟门控**: 无效数据时关闭时钟以降低功耗
4. **背压支持**: 如需支持背压，可添加 ready 信号

2. **IIR 反馈验证**: 重点验证行首/行尾处理
3. **边界条件**: 验证第一行/最后一行的特殊处理
