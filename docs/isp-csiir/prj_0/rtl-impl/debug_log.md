## 问题记录

### 2026-03-23 严重逻辑错误：近似除法右移操作错误

- **现象**: Stage1 梯度计算的近似除以5操作完全错误
- **定位**: `rtl/stage1_gradient.v` 第165行
- **原因**: 实现时对"右移"操作理解错误，取了乘积低位而非截取高位
- **方案**: 修正为 `grad_full >> 10`，即取 `grad_full[GRAD_WIDTH+9:10]`

**错误代码：**
```verilog
wire [GRAD_WIDTH-1:0] grad_sum_comb = grad_full[GRAD_WIDTH-1:0];  // 取低位，错误！
```

**正确代码：**
```verilog
wire [GRAD_WIDTH-1:0] grad_sum_comb = grad_full[GRAD_WIDTH+9:10];  // 右移10位，正确
```

- **状态**: 已修复（2026-03-23）

## 经验教训

### 乘法近似除法的实现检查点

**rtl-impl 必须在实现近似除法时进行以下检查：**

| 检查项 | 检查方法 |
|--------|----------|
| 数学原理 | 写出公式：`a / b ≈ (a × k) >> n`，其中 `k ≈ 2^n / b` |
| 位宽计算 | 乘积位宽 = 原位宽 + 常数位宽 |
| 右移实现 | 截取 `[高位:低位]`，低位 = n（右移位数） |
| 数值验证 | 用具体数值代入验证，确保结果正确 |

### 根因分析（5Why）

```
问题: 近似除法结果完全错误
  └─ Why 1: 代码取了乘积低位而非右移后的结果
      └─ Why 2: 实现者未理解"右移"在定点化中的含义
          └─ Why 3: 没有进行数值验证
              └─ Why 4: 缺乏定点化运算的自检清单
                  └─ Why 5: skill 对定点化运算的检查点不够详细
```

### 强化措施

**rtl-impl skill 必须增加以下检查点：**

1. **所有定点化运算必须进行数值验证**
   - 使用至少2组具体数值代入计算
   - 对比期望值与实际实现值

2. **乘法近似除法必须写出完整推导**
   ```
   目标: a / b
   近似: a × (2^n / b) >> n = a × k >> n
   其中: k = round(2^n / b)
   实现: result[N-1:0] = (a × k)[M-1:n]  // 从第n位开始截取
   ```

3. **位操作必须明确含义**
   - `>> n` = 右移n位 = 截取 `[M-1:n]`
   - `<< n` = 左移n位 = 低位补0
   - 取低位 `data[n-1:0]` ≠ 右移

### 影响范围评估

**此错误影响：**
- Stage1 梯度计算输出完全错误
- Stage3 梯度排序输入错误
- 最终输出像素值错误
- 验证结果必然全部失败

**优先级：最高，立即修复**
